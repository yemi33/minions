# Dallas Learnings — 2026-03-16 (BT-4970959)

## Findings

### Cross-repo PR targeting
- PR-4970959 is in **office-bohemia** (repo ID `74031860-e0cd-45a1-913f-10bbf3f82555`, target `master`), not OfficeAgent. The engine dispatch assumed OfficeAgent but the PR description and ADO metadata clearly point to office-bohemia.
  (source: `az repos pr show --id 4970959` → `targetRefName: refs/heads/master`, `repository.id: 74031860-e0cd-45a1-913f-10bbf3f82555`)

### OfficeAgent worktree created for wrong repo
- A worktree at `C:/Users/yemishin/worktrees/user/yemishin/cowork-shared-tree` was created in the OfficeAgent repo, but this branch only exists in office-bohemia. The OfficeAgent worktree has zero changes vs main.
  (source: `git worktree list` in OfficeAgent, `git diff --stat main` shows 0 files changed)

### office-bohemia bt-4970959 worktree has EPERM issues
- The existing office-bohemia worktree at `C:/Users/yemishin/worktrees/bt-4970959` (detached HEAD at PR commit `47902b5d1630`) has persistent Windows EPERM errors during `yarn install` link step. The `.yarn-state.yml` file is never written, making `yarn workspace` commands fail.
  (source: yarn install output → `Error: EPERM: operation not permitted, unlink ...node_modules\@1js\density\.swc\1c08e27de37d.swcrc`)

### Cherry-pick build strategy works for cross-repo testing
- When a worktree has EPERM issues, cherry-picking the PR commit into an existing working checkout (office-bohemia-cowork) with valid node_modules is a viable workaround for build verification.
  (source: `git cherry-pick FETCH_HEAD --no-commit` followed by `yarn workspace @fluidx/cowork-component build`)

### office-bohemia yarn install warnings are non-fatal
- Yarn 4.12 in office-bohemia produces hundreds of YN0002 (peer dep) and YN0060 (version mismatch) warnings. These cause exit code 1 but the install still completes successfully — node_modules IS populated with 1780+ entries. The key indicator of success is the "Done with warnings" message at the end.
  (source: yarn install output in office-bohemia-cowork)

### Pre-existing Bebop typecheck error
- `apps/bebop/src/features/progression/components/CoworkTranscript/CoworkTranscript.tsx:16` has an unused `FC` import (TS6133). This is pre-existing on the `dallas/add-cowork-route` branch and not introduced by PR-4970959.
  (source: `yarn workspace @bebopjs/bebop build` output)

### Bebop workspace name
- Bebop's package name is `@bebopjs/bebop`, not `bebop`. Use `yarn workspace '@bebopjs/bebop' <cmd>`.
  (source: `apps/bebop/package.json` → `"name": "@bebopjs/bebop"`)

### No tests in new cowork-component package
- PR-4970959 creates `packages/cowork-component/` with schema and adapter code but no jest.config.js or test files. The `yarn workspace @fluidx/cowork-component test` command skips immediately.
  (source: `find packages/cowork-component -name "*.test.*"` returns empty; jest output: "no jest configuration found, skipping jest")

## Patterns

- **office-bohemia build command**: `yarn workspace @fluidx/<pkg> build` runs lage with transpile + typecheck tasks
- **office-bohemia install time**: ~3 minutes on cached run, includes a "MANUAL PATCH PROCESS" step taking ~165s
- **Worktree EPERM workaround**: When Windows file locks prevent yarn install in a worktree, use an existing working checkout + cherry-pick

## Gotchas

- Engine dispatch may route office-bohemia PRs as OfficeAgent tasks — always verify `targetRefName` and repo ID from ADO before building
- Windows Defender / search indexer can cause EPERM during yarn link step in worktrees
- `yarn install` exit code 1 in office-bohemia doesn't necessarily mean failure — check for "Done with warnings" message
