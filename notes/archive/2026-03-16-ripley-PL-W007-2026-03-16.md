### Bebop Design Token System
- Bebop uses `gnrc-*` CSS custom properties for all styling: `--gnrc-color-text-primary`, `--gnrc-color-stroke-neutral-subtle`, `--gnrc-padding-relaxed-small`, `--gnrc-font-size-200`, `--gnrc-border-radius-large`, etc. (source: `apps/bebop/src/features/cowork/components/artifacts/ArtifactPanel.module.css`)

- CSS modules (`.module.css`) are the styling pattern, not inline styles or CSS-in-JS (source: all component files import `styles from './X.module.css'`)

### Jotai Atom Patterns in Bebop
- Write-only atoms use `atom(null, (get, set, payload) => { ... })` pattern for mutations (source: `artifactAtoms.ts:34-58`)
- Derived atoms with fallback: `atom((get) => { ... })` that computes from other atoms (source: `artifactAtoms.ts:13-26`)
- All atom files are standalone — no barrel re-exports (source: directory structure under `atoms/`)

### Artifact Type System
- `ArtifactType = 'docx' | 'pptx' | 'xlsx' | 'pdf' | 'html'` — string union, not enum (source: `types.ts:40`)
- `Artifact` interface has optional `url` field to handle in-progress generations (source: `types.ts:42-47`)
- Only HTML artifacts are previewable in iframe; all binary types get download cards (source: `ArtifactPanel.tsx:18-20`)

### Security Pattern for Agent-Generated Content
- iframe uses `sandbox="allow-same-origin"` (no allow-scripts) to prevent XSS from agent HTML (source: `DocumentPreview.tsx:21`)
- `referrerPolicy="no-referrer"` prevents information leakage (source: `DocumentPreview.tsx:22`)

## Gotchas

### Pre-existing PRs Block `az repos pr create`
- When a PR already exists for a source+target branch pair, `az repos pr create` fails with `TF401179`. Must check first with `az repos pr list --source-branch ... --target-branch ...` before attempting creation. (source: CLI error during this task)

### ADO REST API for PR Comments
- `az repos pr comment` is not a valid command — use REST API directly: `POST .../pullRequests/{id}/threads?api-version=7.1` (source: CLI error `'comment' is misspelled or not recognized`)
- Token must be passed inline to Node.js HTTPS requests; environment variable passing through `node -e` doesn't work with shell-exported vars (source: 302 redirect on first attempt, fixed by string interpolation)

The PR shows all of these. This is a branch stacking pattern — later commits depend on earlier ones. (source: `git log --oneline --all` in worktree)
