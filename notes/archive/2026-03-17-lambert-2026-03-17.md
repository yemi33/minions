### Gotchas

- **Duplicate constants `PERF_EVENT_TYPE` and `INTERACTION_EVENT_TYPE`**: Both set to `'CopilotResultsRender'`. Should be consolidated to a single constant unless intended to diverge. (source: `apps/bebop/src/features/cowork/hooks/useCoworkTelemetry.ts:39-42`)

- **Four unused type definitions in `telemetryTypes.ts`**: `CoworkPerformanceMetrics`, `CoworkInteractionEvent`, `CoworkSessionTelemetry`, `CoworkWebVitalsMetrics` defined but never consumed by the hook. Dead code. (source: `apps/bebop/src/features/cowork/types/telemetryTypes.ts:25-94`)

- **Web vitals type/implementation mismatch**: `CoworkWebVitalsMetrics` declares `lcp`, `cls`, `inp`, `fcp`, `ttfb` but `useCoworkWebVitals` only measures navigation timing (`domContentLoadedEventEnd`, `loadEventEnd`, `responseStart`). Not actual Web Vitals. (source: `telemetryTypes.ts:64-70` vs `useCoworkTelemetry.ts:296-349`)

- **Function identity instability**: All returned functions are plain closures recreated every render. If consumers put in useEffect deps, causes infinite re-renders. Should wrap in useMemo/useCallback. (source: `apps/bebop/src/features/cowork/hooks/useCoworkTelemetry.ts:272-283`)

- **Metadata key collision risk in `trackInteraction`**: `metadata` spread directly into properties can overwrite base dimension keys (`sessionId`, `hostType`, etc.) silently. (source: `apps/bebop/src/features/cowork/hooks/useCoworkTelemetry.ts:218-234`)

- **MCP ADO tools still unavailable**: REST API via curl + GCM Bearer token remains the only working path. (source: ToolSearch returned empty)
