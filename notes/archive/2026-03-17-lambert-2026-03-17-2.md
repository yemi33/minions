### Patterns Discovered

1. **Bebop telemetry hook pattern**: `useCoworkTelemetry` follows established Bebop performance instrumentation conventions — `logToPageTimingsAndTelemetry()` from `dualMetricLogger` for dual logging to `window._pageTimings` and telemetry logger, `useFireOnce` for fire-once Web Vitals collection, `scheduleIdleTask()` for deferred non-critical event logging, `isClientSide` SSR guard as early return. (source: `apps/bebop/src/features/cowork/hooks/useCoworkTelemetry.ts:28-34`)

2. **CopilotResultsRender event type reuse**: All cowork telemetry events (perf metrics, interactions, session events) use `CopilotResultsRender` as eventType for pipeline compatibility with existing Bebop telemetry infrastructure. (source: `apps/bebop/src/features/cowork/hooks/useCoworkTelemetry.ts:42-45`)

3. **useRef for timing state prevents re-renders**: Timing state (`sessionStartMs`, `firstProgressionStepMs`, `agentRequestStartMs`, etc.) stored in `useRef<TimingState>` rather than `useState` to avoid unnecessary component re-renders when timing marks are recorded. (source: `apps/bebop/src/features/cowork/hooks/useCoworkTelemetry.ts:101`)

4. **React Compiler eliminates manual memoization concerns**: Per Bebop AGENTS.md, "Do not add `useCallback`/`useMemo` preemptively." The React Compiler handles memoization automatically, so function objects returned from hooks without `useCallback` is the intended pattern. (source: `apps/bebop/AGENTS.md:39`)

5. **Fire-once hook uses useRef + useEffect pattern**: `useFireOnce` maintains a `hasFired` ref and a `callbackRef` to avoid stale closures while providing single-execution semantics. (source: `apps/bebop/src/core/performance/hooks/useFireOnce.tsx:13-30`)

### Non-Blocking Issues Found

1. **Duplicate constants**: `PERF_EVENT_TYPE` and `INTERACTION_EVENT_TYPE` are both `'CopilotResultsRender'` — should be consolidated to single constant or documented why they are separate. (source: `useCoworkTelemetry.ts:42-45`)

2. **Unused type definitions**: `CoworkPerformanceMetrics`, `CoworkInteractionEvent`, `CoworkSessionTelemetry`, `CoworkWebVitalsMetrics` in `telemetryTypes.ts` are defined but never imported or consumed by the hook implementation. (source: `apps/bebop/src/features/cowork/types/telemetryTypes.ts:32-99`)

3. **Web Vitals type/implementation mismatch**: `CoworkWebVitalsMetrics` declares `lcp`/`cls`/`inp`/`fcp`/`ttfb` but `useCoworkWebVitals` measures `domContentLoaded`, `loadEvent`, and `ttfb` only. The type is not even imported by the hook. (source: `telemetryTypes.ts:72-78` vs `useCoworkTelemetry.ts:305-330`)
