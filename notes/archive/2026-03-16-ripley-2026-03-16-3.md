## Patterns Discovered

### Verification Worktree Pattern
For multi-PR verification:
1. Create detached worktree from main/master: `git worktree add <path> --detach origin/<base>`
2. Merge each PR branch: `git merge origin/<branch> --no-edit`
3. Resolve conflicts preferring PR branch changes
4. Run targeted builds to avoid Docker requirement: `yarn workspace @officeagent/<pkg> build`
5. Use `--mode update-lockfile` before full install when yarn.lock has merge conflicts

### Vite Dev Server Ignores Type Errors
office-bohemia's Bebop uses Vite 7 with esbuild for transpilation. The dev server starts and serves pages even when `tsc --noEmit` reports errors, because esbuild strips types without checking them. This means the cowork route may be navigable even though the build fails typecheck.
(source: Vite starts on port 3000 despite 20 TS errors)

### Pre-existing Issues (Not from PRs)
- OfficeAgent `@officeagent/core` tests have `readyState` readonly assignment errors in WebSocket mock tests (source: `modules/core/tests/websocket/websocket-manager.test.ts:137,342,837,886`)
- office-bohemia Vite dep-scan fails on `@fluidx/odsp-vroom-api` package.json exports (pre-existing, not blocking)

## Gotchas for Future Agents

1. **Never trust `--mode update-lockfile` alone** — it doesn't create `node_modules/.yarn-state.yml`, so builds will fail. Always follow with full `yarn install`.
2. **office-bohemia `yarn install` takes ~20 minutes** — plan accordingly, run in background.
3. **office-bohemia's main branch is `master`**, not `main` — OfficeAgent uses `main`.
4. **Use `yarn workspace <pkg> build` instead of `yarn build --to <pkg>`** — the latter triggers full lage pipeline including Docker image build.
5. **When merging multiple PRs that touch the same files**, check for interface/export compatibility BEFORE committing — TypeScript errors may only appear after all merges complete.
6. **`@fluidx/utilities` may need to be built before bebop** — it's a dependency of featureGates.ts but may not be pre-built in a fresh worktree.

# Multi-PR Verification Worktree Setup

## Steps

1. **Fetch all PR branches**
   ```bash
   cd "<repo-root>"
   git fetch origin "<branch1>" "<branch2>" ... "<base-branch>"
   ```

2. **Create detached worktree from base**
   ```bash
   git worktree add "<worktree-path>" --detach "origin/<base-branch>"
   ```

3. **Merge each PR branch sequentially**
   ```bash
   cd "<worktree-path>"
   git merge "origin/<branch1>" --no-edit
   git merge "origin/<branch2>" --no-edit
   # ... repeat for each branch
   ```

4. **Resolve merge conflicts**
   - For type/interface files: combine exports from both sides
   - For implementation files: prefer the later PR's version (`--theirs`)
   - For lockfiles: take `--theirs` and let yarn install fix it
   - Always commit resolution: `git add -A && git commit -m "Resolve merge: <description>"`

5. **Install dependencies**
   ```bash
   # If yarn.lock has conflicts, update lockfile first:
   powershell.exe -Command "yarn install --mode update-lockfile"
   # Then full install:
   powershell.exe -Command "yarn install"
   ```

6. **Build packages bottom-up** (avoid Docker requirement)
   ```bash
   powershell.exe -Command "yarn workspace @scope/leaf-package build"
   powershell.exe -Command "yarn workspace @scope/dependent-package build"
   ```

7. **Check for integration errors**
   ```bash
   # TypeScript check without emit:
   powershell.exe -Command "yarn workspace @scope/app exec tsc --noEmit"
   ```

## Notes
- Use `--detach` to avoid creating tracking branches in the main repo
- Merge order matters: merge dependency packages before dependent packages
- Each PR may build independently but fail when merged — check after ALL merges
- For large monorepos (227+ packages), `yarn install` can take 15-20 minutes
- Vite/esbuild dev servers may still work despite TS errors (they strip types, don't check them)
