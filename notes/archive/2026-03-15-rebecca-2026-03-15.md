# OfficeAgent Targeted Package Build & Test

## Context
The `yarn build` command runs the full lage pipeline including Docker image build, which fails without Docker Desktop. For library/module changes, use workspace-targeted commands instead.

## Steps

1. **Build the target package:**
   ```powershell
   cd <worktree-path>
   yarn workspace @officeagent/<package-name> build
   ```
   This runs: clean → copyFiles → compileTs → dependency-check (~5s for most packages).

2. **Run tests:**
   ```powershell
   yarn workspace @officeagent/<package-name> test
   ```

3. **Run lint:**
   ```powershell
   yarn workspace @officeagent/<package-name> lint
   ```

4. **Verify downstream consumers** (build a dependent package):
   ```powershell
   yarn workspace @officeagent/core build
   ```
   Core depends on message-protocol, api depends on core — building core verifies no type breakage.

## Notes
- MUST use PowerShell for all yarn commands (bash/sh will fail)
- python3 is unavailable on Windows dev machines — use `node -e` for scripting
- The full `yarn build` or `yarn build --to <pkg>` triggers Docker and will fail without Docker Desktop running
