## Patterns Discovered

### Bebop Route Tree Registration
- New routes require manual update to `apps/bebop/src/routeTree.gen.ts` when building without dev server
- The file is auto-generated by TanStack Router during dev mode but not during `yarn workspace build`
- Must add: import, route update const, FileRoutesByFullPath, FileRoutesByTo, FileRoutesById, FileRouteTypes (fullPaths/to/id unions), MainLayoutRouteChildren interface + object, and FileRoutesByPath declare module
  (source: `apps/bebop/src/routeTree.gen.ts`)

### Bebop Feature Gate Pattern (No Formal SDK)
- Bebop lacks ECS/flight-manager integration; used inline implementation
- Resolution order: query param > localStorage > VITE env var > default false
- Enable via `?EnableBebopCowork=on` or `localStorage.setItem('EnableBebopCowork', 'true')` or `VITE_COWORK_ENABLED=true`
- SSR-safe: checks `typeof window === 'undefined'` to avoid browser API calls during server rendering
  (source: `apps/bebop/src/routes/_mainLayout.cowork.tsx:14-39`)

### Bebop Build Without Dependencies
- `yarn workspace @bebopjs/bebop build` triggers `just-scripts global:build` which runs `yarn lage svgr preTranspile transpile postTranspile typecheck`
- Typecheck fails with 211 pre-existing errors (all `TS2307: Cannot find module '@bebopjs/*'`) because `@bebopjs/*` workspace packages haven't been built
- Transpile step passes (uses Vite/esbuild which handles unbuilt deps differently)
- These are NOT regressions â€” same count on master
  (source: build output from `yarn workspace @bebopjs/bebop build`)

### Workspace Name is `@bebopjs/bebop` Not `bebop`
- Must use full scoped name: `yarn workspace @bebopjs/bebop <command>`
  (source: `apps/bebop/package.json:2`)

### `@fluidx/utilities` Not in Bebop Dependencies
- `packages/utilities/` provides `getSetting` and `getFluidExperiencesSetting` but Bebop's `package.json` doesn't list `@fluidx/utilities` as a dependency
- Cannot import directly; use inline implementation or add as dependency
  (source: `apps/bebop/package.json`)
