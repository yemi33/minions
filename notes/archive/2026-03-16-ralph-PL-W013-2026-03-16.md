**Branch:** user/yemishin/cowork-error-handling

## What Was Built

### 3-Layer Error Handling Architecture
1. **React Error Boundary** (`CoworkErrorBoundary.tsx`): Catches unhandled exceptions in cowork subtree, renders friendly fallback with retry button
2. **Connection Resilience Hook** (`useConnectionResilience.ts`): Manages 6 error scenarios with auto-reconnect, exponential backoff, and state recovery
3. **Status Banner** (`ConnectionStatusBanner.tsx`): User-facing inline banner with error-specific messages and action buttons

### Connection State Machine
- `connectionAtoms.ts` tracks: phase (idle/connected/reconnecting/error/retrying), reconnect attempts, rate limit info
- String union types (not enums) for Vite/esbuild tree-shaking (source: `apps/bebop/src/features/cowork/atoms/connectionAtoms.ts:5-10`)
- Readonly fields on all interfaces for immutability (source: `connectionAtoms.ts:12-20`)

### Error Scenarios
| Error | Detection | Recovery |
|-------|-----------|----------|
| Container unavailable | 503 status | Retry button |
| WebSocket disconnect | Connection lost | Auto-reconnect, exponential backoff (1s-30s) + jitter (±25%) |
| Agent timeout | Configurable (120s default) | Notification + cancel button |
| Token expiry | 401 status | Transparent refresh via `reportTokenRefreshed()` |
| Rate limiting | 429 status | Queue position + ETA display, auto-retry |
| Transient failures | Other errors | Auto-reconnect |

### Lage Build System
- `yarn workspace @bebopjs/bebop build` runs transpile + typecheck, but typecheck fails without dependencies built first (source: lage.config.js:79-87 — typecheck depends on `^typecheck` and buildTasks)
- `yarn lage transpile --to @bebopjs/bebop` is the fast validation path — proves code compiles without building all 200+ packages (source: successful 0.36s transpile)
- There's no `build` task in lage pipeline — use `transpile`, `typecheck`, `lint` etc. directly (source: lage.config.js pipeline definition)

### CSS Module Pattern
- All CSS in `@layer application {}` wrapper (source: `CoworkErrorBoundary.module.css:1`)
- Use Bebop design tokens: `--gnrc-*` variables for colors, spacing, typography (source: `CoworkErrorBoundary.module.css:9-31`)

### React Compiler Compatibility
- No manual `useCallback`/`useMemo` — React Compiler handles memoization in Bebop (React 19 + React Compiler) (source: comment in `useConnectionResilience.ts:54`)
- Functions created in hooks can be used in `useEffect` dependency arrays safely
