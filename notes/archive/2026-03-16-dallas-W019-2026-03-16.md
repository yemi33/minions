## Patterns Discovered

### CSS ESLint disable comments in office-bohemia
- **ESLint for CSS uses `/* eslint-disable-next-line rule-name */` syntax** — must be on the line BEFORE the flagged property, inside the CSS rule block. (source: `apps/bebop/eslint.config.mjs`, verified by lint pass)
- **`font-family: inherit` cannot have fallbacks** — `inherit` is a CSS-wide keyword that must be the sole value. `inherit, sans-serif` triggers `css/no-invalid-properties`. The correct fix is an eslint-disable comment. (source: lint run 1, css/no-invalid-properties error)
- **`word-break: break-word` is non-baseline** — Replace with `overflow-wrap: break-word` which is baseline and functionally equivalent. (source: css/use-baseline rule)
- **`resize` is flagged by css/use-baseline** — Despite being widely supported, it's not "newly available baseline". Use eslint-disable comment. (source: CoworkChatPanel.module.css:88)

### React 19 deprecated imports in Bebop
- **`FormEvent` from 'react' triggers import/no-deprecated** in office-bohemia's React 19 + TypeScript 5.9 setup. Fix: use inline type `{ preventDefault(): void }` when the event is optional and only used for preventDefault. (source: CoworkChatPanel.tsx:6,18)
